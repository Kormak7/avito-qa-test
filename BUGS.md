# Найденные баги:

## 1) TC-06. При создании нового объявления можно изменить регистр в наименовании параметров запроса
Шаги воспроизведения:
1. Отправить POST /api/1/item
2. В body передать sellerId вместо sellerID
3. Проверить ответ API

Ожидаемый результат:
API возвращает 400 Bad Request, объявление не создаётся.

Фактический результат:
API возвращает 200 OK, объявление успешно создаётся.

Серьёзность: High

---

## 2) TC-10. При создании нового объявления можно ввести отрицательную цену

## 4) TC-16. Отсутствует ограничение на количество символов для параметра name при создании объявления
Шаги воспроизведения:
1. Отправить POST /api/1/item
2. Передать name длиной 1000 символов
3. Проверить ответ API

Ожидаемый результат:
API либо ограничивает длину поля и возвращает 400 Bad Request, либо в документации явно зафиксировано отсутствие лимита.

Фактический результат:
API принимает значение длиной 1000 символов без явного ограничения.

Серьёзность: Medium

---
