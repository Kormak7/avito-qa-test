# Тесткейсы:

## 1) Успешное создание объявления с валидными данными
Предусловия:
- Сервис доступен.
- Выбран уникальный `sellerID` из допустимого диапазона, например `111111`.

Тестовые данные:
- `sellerID = 111111`
- `name = "iPhone 13"`
- `price = 65000`
- `statistics = {"likes": 10, "viewCount": 150, "contacts": 3}`

| Шаги: | Ожидаемый результат: |
|--|--|
|1. Отправить `POST /api/1/item` с валидным JSON-телом.|Сервер возвращает `200 OK`.|
|2. Проверить тело ответа.|В ответе присутствуют `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`.|
|3. Сверить значения полей ответа с отправленными данными.|`sellerId`, `name`, `price`, `statistics` соответствуют запросу. `id` уникален, `createdAt` заполнен.|
|4. Сохранить `id` созданного объявления.|`id` можно использовать для последующих запросов.|

## 2) Создание объявления с одинаковыми данными дважды
Предусловия:
- Сервис доступен.
- Подготовлен валидный JSON для создания объявления.

Тестовые данные:
- Один и тот же набор данных из кейса 1.

| Шаги: | Ожидаемый результат: |
|--|--|
|1. Отправить `POST /api/1/item` с валидным телом.|Создано объявление, сервер возвращает `200 OK`.|
|2. Отправить тот же `POST /api/1/item` повторно без изменений.|Создано второе объявление, сервер снова возвращает `200 OK`.|
|3. Сравнить `id` и `createdAt` двух ответов.|Значения `id` различаются, `createdAt` также различается или соответствует времени создания.|
|4. Сравнить остальные поля двух ответов.|`sellerId`, `name`, `price`, `statistics` одинаковые.|
